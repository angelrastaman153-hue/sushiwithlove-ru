## 1. Цель и контекст

### Цель сайта

Конвертировать посетителя в покупателя — максимально коротким путём от первого взгляда до оформленного заказа, который автоматически попадает в систему Frontpad.

**Измеримые цели:**
- Конверсия в заказ: ≥ 5% с холодного трафика, ≥ 15% с тёплого
- Средний чек: ≥ 900 ₽
- Время до первого добавления в корзину: ≤ 60 секунд
- Скорость загрузки: LCP ≤ 2.5 секунды

### Что это за проект

Одностраничный сайт-лендинг для суши-бара в Кургане. Сайт — основной канал онлайн-заказов. Заказы из корзины уходят напрямую в Frontpad (система управления доставкой), оператор видит их в своём интерфейсе мгновенно.

**Сайт не является:** агрегатором, мультистраничным каталогом, приложением с личным кабинетом (в MVP).

## 2. Целевая аудитория

### Основной сегмент — «Домашний заказчик»

| Параметр | Описание |
|---------|---------|
| Возраст | 22–42 года |
| Устройство | Смартфон (65–70% трафика), десктоп (30–35%) |
| Мотивация | Ужин дома, не хочется готовить, пятничный вечер |
| Боль | «Не знаю, доставят ли ко мне», «Долго ждать», «Рыба несвежая» |
| Решение | Быстрый выбор из меню → оплата онлайн → ждёт звонка |

### Второй сегмент — «Корпоратив / Компания»

| Параметр | Описание |
|---------|---------|
| Возраст | 25–45 лет, принимает решение за группу |
| Мотивация | Заказ на офис, день рождения, посиделки |
| Боль | «Нужно много, нужно дёшево, нужно быстро» |
| Решение | Сеты → прогресс-бар до бесплатной доставки → крупный заказ |

### Третий сегмент — «Первый раз»

Пришёл из таргета или по рекомендации. Не знает бренд. Нужно: социальное доказательство + скидка на первый заказ + простота оформления.

```

```

### Подзаголовок

```
Охлаждённая рыба каждый день — без заморозки. Минимальный заказ 600 ₽.
```

### Акционный бейдж (hero)

```

```

### Позиционирование

Не самый дешёвый и не «вау-премиум». **Средний+ сегмент**: качественная еда, своевременная доставка, удобный сайт. Отстройка от конкурентов через: свежесть рыбы + скорость + удобство заказа (не надо звонить).

### Слоган (дополнительно)

```

```

### ШАПКА (Header · sticky)

**Поведение:** фиксируется при скролле. На тёмном фоне с лёгким blur (glassmorphism).

| Элемент | Контент | Поведение |
|---------|---------|----------|
| Логотип | SVG-лого «Sushi with Love» | Клик → якорь #hero |
| Навигация | Меню · Акции · Доставка · О нас | Плавный скролл к секции |
| Телефон | +7 (XXX) XXX-XX-XX | `<a href="tel:+7...">` |
| Время работы | Пн–Вс 10:00–22:00 | Статичный текст |

**Мобиле:** скрыть навигацию в бургер-меню. Телефон + корзина всегда видны.

### 01. HERO

**Цель:** удержать пользователя за 3 секунды, дать ему причину остаться.

**Фон:** полноэкранное фото роллов на тёмном фоне — собственная съёмка, не стоковое. Альтернатива: короткое видео (15 сек loop) — шеф нарезает лосося, скатывает ролл, подаёт тарелку. На мобиле — статичное фото.

**Контент:**

```
[БЕЙДЖ]  −15% на первый заказ · Промокод ПЕРВЫЙ

[H1]  Суши и роллы
      с доставкой за 45 минут

[P]   Охлаждённая рыба каждый день.
      Минимальный заказ 600 ₽ · Курган

[CTA] [ Смотреть меню → ]

[ПОДСКАЗКА]  ↓ Листайте, чтобы выбрать блюда
```

**Дополнительно (правый угол / overlay):**
```
⏱ Акция до 23:59
Сет «Вечерний» −20%
[Таймер: 04:32:17]
```

**Технические требования:**
- Высота: 100vh на десктопе, 85vh на мобиле
- Кнопка CTA не уходит за fold ни на одном устройстве
- Видео: `autoplay muted loop playsinline`, замена на poster при `prefers-reduced-motion`

### 04. МЕНЮ

**Цель:** дать пользователю выбрать и добавить блюда максимально быстро.

#### Sticky-навигация по категориям

```
[ Популярное ] [ Роллы ] [ Сеты ] [ Суши ] [ WOK ] [ Напитки ] [ Соусы ]
```

Прилипает к шапке при скролле (`position: sticky; top: 64px`). Активный таб подсвечен акцентным цветом. Клик — плавный скролл к разделу.

#### Карточка блюда

```
┌──────────────────────────────────┐
│  [Фото 4:3, нейтральный фон]    │
│                          [🔥ХИТ] │
├──────────────────────────────────┤
│  Филадельфия классик             │
│  Лосось · крем-сыр · огурец     │
│  8 шт · 260 г                   │
├──────────────────────────────────┤
│  590 ₽              [ + Добавить]│
└──────────────────────────────────┘
```

**Обязательные поля карточки:**
- Фото (собственное, не стоковое)
- Название (по-русски, читаемо)
- Краткий состав (через точку, максимум 5 ингредиентов)
- Вес и количество штук
- Цена
- Кнопка «+ Добавить» → при повторном нажатии превращается в счётчик `[− 1 +]`
- Бейдж: «Хит», «Новинка», «Острое 🌶», «Нет в наличии» (если стоп-лист)

**Бейджи категорий (заголовки разделов внутри страницы):**
```
── Популярное ──────────────────
[карточки]

── Роллы ───────────────────────
[карточки]
```

#### Toast при добавлении

```
┌──────────────────────────────────┐
│  ✓  Филадельфия добавлена        │
│     Корзина: 3 позиции · 1 180 ₽ │
│              [ Оформить заказ → ] │
└──────────────────────────────────┘
```

Появляется снизу-справа. Автоскрытие через 3 секунды. Не открывает drawer автоматически.

## 6. Дизайн-система

### 6.1 Цветовая палитра

| Токен | HEX | Применение |
|-------|-----|-----------|
| `--color-bg` | `#0F0B08` | Основной фон (тёплый почти-чёрный) |
| `--color-surface` | `#1E1810` | Карточки, панели |
| `--color-surface-2` | `#2A2218` | Hover-состояние карточек |
| `--color-accent` | `#E8A847` | CTA-кнопки, активные элементы, акценты |
| `--color-accent-hover` | `#D4923A` | Hover на accent |
| `--color-danger` | `#C84B31` | Цены, badges «Акция», таймер |
| `--color-text-primary` | `#F5F0E8` | Основной текст |
| `--color-text-secondary` | `#A89880` | Вспомогательный текст, подписи |

## 7. Обязательные элементы и поведение

### 7.1 Шапка (Header)

- `position: sticky; top: 0; z-index: 100`
- Glassmorphism-фон при скролле > 50px
- Высота: 64px десктоп / 56px мобиле
- Иконка корзины с бейджем количества (обновляется в реальном времени)
- Телефон кликабелен (`tel:`) на всех устройствах

### 7.2 Корзина

**Десктоп — slide-out drawer (справа):**
- Ширина: 420px
- Overlay: `rgba(0,0,0,0.5)` на фон
- Открытие: по клику на иконку корзины в шапке
- Закрытие: клик вне drawer, кнопка ✕, Escape

**Мобиле — sticky bottom bar + bottom sheet:**
```
Bottom bar (всегда visible, скрыта если корзина пуста):
  position: fixed; bottom: 0; left: 0; right: 0
  height: 64px
  background: var(--color-accent)
  padding-bottom: env(safe-area-inset-bottom)
  content: [🛒 3 позиции  ·  1 180 ₽  →  Оформить]

Bottom sheet (при клике на bar):
  position: fixed; bottom: 0; width: 100%
  border-radius: 20px 20px 0 0
  max-height: 90vh; overflow-y: auto
  drag handle сверху
  закрытие: swipe down > 100px или клик на backdrop
```

**Содержимое корзины:**
1. Список позиций с фото, именем, счётчиком количества `[− N +]`, ценой, кнопкой удаления
2. Upsell-строка: «Часто добавляют: [Васаби 50₽ +] [Имбирь 30₽ +] [Соус Унаги 60₽ +]»
3. Промокод: `[__________________] [Применить]`
4. Прогресс-бар: `████████░░ До бесплатной доставки ещё 220 ₽`
5. Итоговая сумма с разбивкой
6. Кнопка «Оформить заказ» — всегда видна, не уходит за fold

**Сохранение в `localStorage`** — корзина не сбрасывается при перезагрузке. TTL: 24 часа.

### 7.3 Навигация по меню

- Категории — горизонтальный sticky-бар под шапкой
- `scroll-snap-type: x mandatory` на мобиле
- Active-состояние категории определяется через IntersectionObserver
- При клике на категорию — `scrollIntoView({ behavior: 'smooth' })`
- `scrollbar-width: none` — нет видимого скроллбара

### 7.4 Форма заказа — поля и валидация

| Поле | Тип | Валидация | Frontpad-параметр |
|------|-----|----------|------------------|
| Имя | text | Непустое, ≥ 2 символа | `name` |
| Телефон | tel | Маска +7, 11 цифр | `phone` (7XXXXXXXXXX) |
| Улица | text + DaData | Непустое, выбрано из подсказок | `street` |
| Дом | text | Непустое | `home` |
| Квартира | text | Опционально | `apart` |
| Подъезд | text | Опционально | `pod` |
| Этаж | text | Опционально | `et` |
| Время | datetime | Слоты минимум +60 мин от текущего | `datetime` |
| Оплата | radio | Один из трёх вариантов | `pay` (1/2/3) |
| Персон | number | 1–20 | `person` |
| Комментарий | textarea | Опционально | `descr` |

**Валидация:** inline, сразу после `blur` поля. Не после нажатия кнопки. Ошибка подсвечивает поле красным + текст под полем.

**После успешной отправки:** не сбрасывать форму до получения ответа от Frontpad. При ошибке API — показать `«Не удалось оформить заказ. Позвоните нам: +7...»`, форму не сбрасывать.

### 7.5 Анимации (обязательные)

| Элемент | Анимация | Технология |
|---------|---------|------------|
| Появление секций при скролле | fade + slide-up (y: 30px → 0, opacity: 0 → 1) | GSAP + ScrollTrigger |
| Счётчики в «Полосе доверия» | animate number 0 → N | GSAP ScrollTrigger |
| Карточки меню | stagger appearance (задержка 0.05s между) | GSAP |
| Кнопка «+» в карточке | morph → счётчик | CSS transition |
| Toast уведомление | slide-in снизу, fade-out | CSS animation |
| Bottom sheet | translateY(100%) → 0 | CSS transition + touch |
| Drawer корзины | translateX(100%) → 0 | CSS transition |
| Таймер акции | живой отсчёт | setInterval |
| Прогресс-бар корзины | width анимируется при изменении суммы | CSS transition |
| Hover карточки | translateY(-4px) + glow | CSS transition |
| Число в бейдже корзины | scale(1.3) → scale(1) при изменении | CSS keyframe |

**`prefers-reduced-motion: reduce`** — все анимации отключаются или сводятся к мгновенным переходам.

## 8. Интеграция с Frontpad

### 8.1 Архитектура (обязательно через бэкенд)

```
Браузер (Next.js frontend)
       ↓  POST /api/order
Next.js API Route (бэкенд-прокси)
       ↓  POST с secret key
Frontpad API
       ↓  { order_id, status }
Next.js API Route
       ↓  { order_id, status }
Браузер → показать подтверждение
```

**Secret key хранится исключительно в `.env.local` на сервере. Никогда не попадает в браузер.**

### 8.2 Основные API-вызовы

**Получение каталога (при сборке / ISR):**
```
GET https://app.frontpad.ru/api/new/products?secret=KEY
GET https://app.frontpad.ru/api/new/categories?secret=KEY
```
Кешировать: ISR revalidate = 900 (15 мин). В стоп-листе — скрывать карточку или показывать «Нет в наличии».

**Создание заказа:**
```
POST https://app.frontpad.ru/api/new/order
```
Обязательные поля: `secret`, `product[]`, `product_kol[]`, `name`, `phone`.
Дополнительные: `street`, `home`, `apart`, `et`, `pod`, `pay`, `datetime`, `descr`, `person`, `certificate`, `self`.

**Формат ответа:**
```json
{ "order_id": "12345", "order_number": "А-101", "status": 1 }
```
При `status !== 1` или сетевой ошибке — показать сообщение с телефоном, не сбрасывать форму.

**Бонусы клиента (по номеру телефона):**
```
POST https://app.frontpad.ru/api/new/score { secret, phone }
```
Показывать баланс бонусов в форме после ввода телефона.

**Зоны доставки:**
```
GET https://app.frontpad.ru/api/new/delivery?secret=KEY
```
Возвращает полигоны. Использовать для валидации адреса + отображения карты. Кеш: 1 час.

### 8.3 Обработка ошибок

| Ситуация | Действие |
|---------|---------|
| Frontpad недоступен | Показать «Оформите заказ по телефону +7...», не ломать сайт |
| Товар в стоп-листе | Автоматически убрать из карточек при следующем ISR |
| Неверный формат данных | Валидация на фронте не пропустит |
| Дублирующийся заказ | Показать предупреждение «Заказ уже оформлен» |
| Ошибка оплаты ЮКасса | Предложить другой способ оплаты, не терять корзину |

## 13. Telegram Mini App — план разработки v1.0

> Дата: 2026-04-13
> Основан на: research.md § 8 «Экспертная оценка Telegram Mini App»
> Статус: **черновик плана, v1**

### Концепция

Mini App — дополнительный канал к лендингу. Открывается из Telegram-бота одной кнопкой. Выглядит как нативное приложение: нет адресной строки, нет кнопок браузера, нативные компоненты Telegram. Цель — чтобы постоянные клиенты заказывали не через сайт, а прямо из Telegram за 30–60 секунд.

**Принцип:** минимум экранов, максимум скорости. Пользователь пришёл голодный — не заставляй его разбираться.

---

### Навигационная модель

```
Шапка Telegram (BackButton слева, название по центру)
─────────────────────────────────────────
  [Контент экрана]
─────────────────────────────────────────
MainButton (нативная кнопка Telegram внизу)
```

- **BackButton** (`tg.BackButton`) — возврат назад или закрытие Mini App
- **MainButton** (`tg.MainButton`) — главное действие текущего экрана
- **Bottom sheet** — второстепенный контент без смены экрана
- Никаких кастомных шапок — только контент и нативные элементы Telegram

---

### Список экранов

```
[Загрузка] → [Главная: Меню]
                   │
                   ├── тап на карточку → [Bottom sheet: Товар]
                   │
                   └── MainButton «Корзина (N) · X ₽»
                              │
                         [Bottom sheet: Корзина]
                              │
                         MainButton «Оформить»
                              │
                         [Экран: Адрес доставки]
                              │
                         MainButton «Далее»
                              │
                         [Экран: Детали заказа]
                              │
                         MainButton «Оформить заказ»
                              │
                         [Экран: Заказ принят]
                              │
                         Кнопка «Закрыть» → tg.close()
```

Итого: **5 состояний** (загрузка + главная + адрес + детали + подтверждение) + **2 bottom sheet** (товар, корзина).

---

### Экран 0 — Загрузка

**Что видит пользователь:**
- Логотип «Sushi with Love» по центру (SVG, анимация появления)
- Надпись «Загружаем меню...» под логотипом
- Цвет фона из `tg.themeParams.bg_color` — чтобы не было белой вспышки

**Действия:** нет (автоматический переход)

**Переход:** через 1–2 сек после загрузки данных меню → Экран 1

**Технически:**
```js
tg.expand();                         // Развернуть на весь экран
tg.disableVerticalSwipes();          // Отключить pull-to-close
document.body.dataset.theme = tg.colorScheme; // dark | light
```

---

### Экран 1 — Главная: Меню

**Главный экран приложения. Пользователь проводит здесь 90% времени.**

#### Шапка (sticky, высота 56px)

| Элемент | Детали |
|---------|--------|
| Логотип | SVG, 32px, слева |
| Название | «Sushi with Love», по центру, 16px medium |
| Иконка корзины | Справа, 28px, с бейджем-счётчиком (красный круг с цифрой) |

#### Таб-бар категорий (sticky под шапкой, горизонтальный скролл)

Категории из меню суши-бара:

```
[Хиты] [Сеты] [Роллы] [Горячие] [Акции] [Напитки] [Добавки]
```

- Активная категория: акцентный цвет + нижнее подчёркивание
- При скролле страницы: активный таб автоматически меняется (Scrollspy / IntersectionObserver)
- Тап на таб: плавный скролл к разделу

#### Карточки товаров (2 колонки, gap 12px)

Каждая карточка:

| Элемент | Размер / детали |
|---------|----------------|
| Фото | 100% ширины карточки, aspect-ratio 4:3, object-fit: cover, border-radius 12px |
| Бейдж | Абсолютно поверх фото: «Хит» (красный), «Новинка» (зелёный), «Акция» (оранжевый) |
| Название | 14px medium, 2 строки максимум, ellipsis |
| Состав | 12px, серый, 1 строка, ellipsis |
| Вес | 12px, серый («260 г · 8 шт») |
| Цена | 16px bold, акцентный цвет |
| Кнопка | Круглая «+» 44×44px, акцентный цвет — или счётчик «−  1  +» если уже добавлен |

**Действия на карточке:**
- Тап на фото или название → открыть Bottom sheet: Товар
- Тап на «+» → добавить в корзину (haptic light + анимация счётчика)
- Тап на «−» / «+» в счётчике → изменить количество (haptic light)

#### MainButton (нативная кнопка Telegram)

- Скрыта если корзина пуста
- При добавлении первого товара: плавно появляется с анимацией
- Текст: `«🛒 Корзина · 3 товара · 890 ₽»`
- Цвет: `tg.themeParams.button_color`
- Тап → открыть Bottom sheet: Корзина

---

### Bottom sheet — Товар

**Открывается поверх Меню, не меняет экран.**

| Элемент | Детали |
|---------|--------|
| Фото | Полная ширина, aspect-ratio 16:9, border-radius 16px вверху |
| Бейдж | Если есть («Хит», «Акция») |
| Название | 20px bold |
| Состав | 14px, с переносом строк (полный текст) |
| Вес / КБЖУ | 13px, серый — в одну строку: «260 г · 8 шт · 320 ккал» |
| Цена | 22px bold, акцентный цвет |
| Кнопка «В корзину» | Полная ширина, 52px высота, MainButton-стиль |
| Кнопка «−  2  +» | Если уже добавлен — вместо «В корзину» |
| Блок «Часто берут вместе» | 3 мини-карточки горизонтально, кнопка «+ добавить» |

**Действия:**
- Свайп вниз или тап вне sheet → закрыть
- Тап «В корзину» → добавить, haptic success, закрыть sheet
- Тап на мини-карточку в «часто берут» → добавить в корзину (haptic light)

**BackButton:** показать, тап → закрыть sheet

---

### Bottom sheet — Корзина

**Открывается из MainButton или иконки корзины в шапке.**

#### Прогресс-бар до бесплатной доставки (если не достигнут порог)

```
[██████░░░░] До бесплатной доставки: ещё 210 ₽
```

#### Список товаров

Каждая строка:

| Элемент | Детали |
|---------|--------|
| Фото | 56×56px, border-radius 8px |
| Название | 14px medium, 1 строка |
| Цена за штуку | 13px, серый |
| Счётчик «−  1  +» | Кнопки 36×36px минимум |
| Итог строки | 14px bold, справа |
| Кнопка удаления | Иконка корзины/крестика, 36×36px |

#### Блок промокода

```
[Введите промокод ________] [Применить]
```

- При успехе: зелёная галочка + строка скидки в итоге
- При ошибке: красный крестик + подпись «Промокод не найден»

#### Итоговый блок

```
Товары:        890 ₽
Доставка:      100 ₽   (или «Бесплатно»)
Скидка:       −89 ₽    (если применён промокод)
──────────────────────
Итого:         901 ₽
```

**Действия:**
- Изменение количества → пересчёт суммы в реальном времени
- Свайп строки влево → удалить товар (с подтверждением `tg.showConfirm`)
- Свайп sheet вниз → закрыть

**MainButton:** `«Оформить заказ · 901 ₽»` → переход на Экран 2

---

### Экран 2 — Адрес доставки

**Минималистичный экран, максимум автоматизации.**

#### Переключатель вверху

```
[  Доставка  ]  [  Самовывоз  ]
```

Высота 44px, сегментированный контрол.

#### Блок «Доставка» (активен по умолчанию)

| Элемент | Детали |
|---------|--------|
| Кнопка «Определить адрес» | 52px, иконка геолокации, полная ширина |
| Поле улицы | `type="text"`, autocomplete=street-address, подсказки DaData |
| Поле дома | `type="text"`, рядом |
| Поле квартиры | `type="text"`, опционально, подпись «необязательно» |
| Зона доставки | После ввода адреса — автоматически: «Зона: Центр · Доставка бесплатна» или «Зона: Заречный · +100 ₽» |

#### Блок «Самовывоз»

| Элемент | Детали |
|---------|--------|
| Адрес точки | Улица, номер, крупно |
| Схема / карта | Статичная картинка или ссылка «Открыть в картах» |
| Время готовности | «Готовим ~30 минут» |

**Действия:**
- Тап «Определить адрес» → `tg.requestLocation()` → автозаполнение
- Ввод адреса → live-подсказки DaData → выбор из выпадающего списка
- После выбора адреса: показать зону и стоимость доставки

**MainButton:** «Далее» (активна только если адрес заполнен)
**BackButton:** вернуться к Корзине

---

### Экран 3 — Детали заказа

**Финальный шаг перед отправкой. Не более 4 полей.**

| Поле | Тип | Детали |
|------|-----|--------|
| Имя | `type="text"`, autocomplete=given-name | Предзаполнено из `tg.initDataUnsafe.user.first_name` |
| Телефон | Кнопка «Поделиться номером» | `tg.requestContact()` — 1 нажатие вместо ввода |
| Способ оплаты | Радио-группа: «Наличные» / «Картой курьеру» / «Онлайн» | Иконки + подписи, карточки-выборы |
| Комментарий | `type="text"`, опционально | «Без острого, позвоните в домофон...» |
| Палочки | Числовой степпер «−  0  +» | Предзаполнен 0 |

#### Итоговый блок (показывается внизу, над MainButton)

```
Ваш заказ:   3 позиции · 901 ₽
Доставка:    ул. Ленина, 55, кв. 12
Время:       ~45 минут
```

**Действия:**
- Тап «Поделиться номером» → нативный диалог Telegram → автозаполнение поля
- Выбор оплаты → визуальный выбор карточки
- Изменение способа оплаты → пересчёт (если онлайн — другая сумма/логика)

**MainButton:** «Оформить заказ · 901 ₽» (активна если имя + телефон заполнены)
**BackButton:** вернуться к Экрану 2 (адрес)

---

### Экран 4 — Заказ принят

**Финальный экран. Снимает тревогу, благодарит, закрывает сессию.**

| Элемент | Детали |
|---------|--------|
| Анимация | Лоти-анимация или CSS: галочка в круге + confetti (лёгкая) |
| Заголовок | «Заказ принят!» — 24px bold |
| Подзаголовок | «Доставим к ~14:35. Позвоним для подтверждения.» |
| Номер заказа | «Заказ #4821» — серый, 14px |
| Состав заказа | Краткий список: «3 позиции · 901 ₽» |
| Кнопка «Отлично» | 52px, полная ширина, акцентный цвет → `tg.close()` |
| Кнопка «Поделиться» | Второстепенная, под основной → `tg.switchInlineQuery()` с реферальной ссылкой |

**Haptic:** `tg.HapticFeedback.notificationOccurred('success')` при показе экрана

**Действия:**
- «Отлично» → закрыть Mini App (`tg.close()`)
- «Поделиться» → открыть диалог выбора чата в Telegram с сообщением о суши-баре

**BackButton:** скрыт на этом экране (заказ уже оформлен, нет смысла возвращаться)

### Дизайн-система Mini App

#### Цвета — полностью из Telegram

```css
:root {
  --tg-bg:          var(--tg-theme-bg-color);
  --tg-text:        var(--tg-theme-text-color);
  --tg-hint:        var(--tg-theme-hint-color);
  --tg-link:        var(--tg-theme-link-color);
  --tg-btn:         var(--tg-theme-button-color);
  --tg-btn-text:    var(--tg-theme-button-text-color);
  --tg-secondary:   var(--tg-theme-secondary-bg-color);
  --tg-header:      var(--tg-theme-header-bg-color);

  /* Кастомные акценты (поверх темы) */
  --accent:         #E8553E;   /* тёплый красный для еды, работает в обеих темах */
  --accent-light:   rgba(232, 85, 62, 0.12);
  --success:        #4CAF50;
  --badge-hit:      #E8553E;
  --badge-new:      #4CAF50;
  --badge-sale:     #FF9500;
}
```

#### Типографика

| Роль | Размер | Вес |
|------|--------|-----|
| Заголовок экрана | 20px | 700 |
| Название товара | 14–16px | 600 |
| Цена | 16–22px | 700 |
| Описание / состав | 12–13px | 400 |
| Кнопки | 15–16px | 600 |
| Подписи | 12px | 400 |

Шрифт: системный стек (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`) — как у самого Telegram.

#### Размеры интерактивных элементов

| Элемент | Минимум |
|---------|---------|
| Кнопки «+», «−» | 44×44px |
| MainButton | 52px высота |
| Табы категорий | 44px высота |
| Строки списка | 64px высота |
| Карточки товаров | 160px минимальная высота |
| Область тапа на строке корзины | весь ряд, 64px |

#### Анимации

| Событие | Анимация |
|---------|----------|
| Добавление в корзину | Иконка «прыгает» + счётчик пульсирует (scale 1→1.3→1) |
| Открытие bottom sheet | Slide up, 300ms ease-out |
| Закрытие bottom sheet | Slide down, 250ms ease-in |
| Появление MainButton | Fade + slide up, 200ms |
| Загрузка карточек | Skeleton-плейсхолдеры (shimmer) |
| Экран «Заказ принят» | Scale up + confetti, 500ms |

### Технический стек

```
Telegram WebApp SDK   — telegram-web-app.js (1 скрипт, ~50KB)
HTML/CSS/JS           — адаптация текущего index.html
tg.CloudStorage       — корзина, последний заказ, промокоды
tg.requestContact     — получение телефона
tg.requestLocation    — геолокация для адреса
Frontpad API          — отправка заказа (та же интеграция, что на сайте)
Bot API               — уведомления о статусе заказа
DaData API            — подсказки адресов (уже используется на сайте)
Хостинг              — Vercel (уже задеплоен, HTTPS есть)
```

#### Инициализация (добавить в `<head>`)

```html
<script src="https://telegram.org/js/telegram-web-app.js"></script>
```

#### Стартовый JS

```js
const tg = window.Telegram.WebApp;

tg.ready();
tg.expand();
tg.disableVerticalSwipes();

// Тема
document.documentElement.setAttribute('data-theme', tg.colorScheme);
tg.onEvent('themeChanged', () => {
  document.documentElement.setAttribute('data-theme', tg.colorScheme);
});

// Персонализация
const user = tg.initDataUnsafe?.user;
if (user?.first_name) {
  document.querySelector('.greeting').textContent = `Привет, ${user.first_name}!`;
}
```

### Что НЕ будет в первой версии

| Функция | Почему отложена |
|---------|----------------|
| Онлайн-оплата через Telegram Payments | Требует подключения эквайринга, отдельная интеграция |
| Трекер курьера на карте | Нужен реальный GPS от курьера, сложная инфраструктура |
| Программа лояльности / баллы | Требует бэкенд с базой пользователей |
| История заказов | Нужен бэкенд, авторизация |
| Личный кабинет | То же |
| Отзывы / рейтинг товаров | Требует модерацию |
| Поиск по меню | Отложить до версии 1.1 |
| Push-рассылки (акции, новинки) | Настраивается в боте отдельно |
| Конструктор ролла / кастомизация | Сложная логика, MVP без неё |
| Мульти-ресторан (несколько точек) | У нас одна точка |

### Приёмочные критерии v1.0

#### Функциональность
- [ ] Mini App открывается из бота и разворачивается на весь экран
- [ ] Меню загружается, все категории отображаются
- [ ] Товар можно добавить в корзину / изменить количество / удалить
- [ ] Промокод применяется и пересчитывает сумму
- [ ] Прогресс-бар до бесплатной доставки работает корректно
- [ ] Адрес доставки принимается (ручной ввод + геолокация)
- [ ] Телефон получается через `tg.requestContact()` одним нажатием
- [ ] Заказ отправляется в Frontpad с правильными данными
- [ ] Экран «Заказ принят» показывает время и номер заказа
- [ ] Корзина сохраняется в `tg.CloudStorage` и восстанавливается при повторном открытии

#### UX / внешний вид
- [ ] Тёмная и светлая тема работают корректно (нет белых вспышек, нет инвертированных цветов)
- [ ] Все интерактивные элементы минимум 44×44px
- [ ] BackButton показывается на всех экранах кроме главного и финального
- [ ] Haptic feedback работает на iOS при добавлении в корзину
- [ ] Нет горизонтальной прокрутки
- [ ] Skeleton-плейсхолдеры показываются при загрузке меню
- [ ] На Android и iOS нет визуальных артефактов с `position:fixed`

#### Производительность
- [ ] Первая загрузка < 3 секунд на мобильном 4G
- [ ] JS-бандл < 200KB gzip
- [ ] Изображения в WebP, < 120KB каждое, lazy loading

### Порядок разработки

```
Этап 1 — Каркас (1–2 дня)
  ├── Подключить telegram-web-app.js
  ├── Инициализация: expand, disableVerticalSwipes, тема
  ├── CSS-переменные из tg.themeParams
  └── Статичная вёрстка главного экрана (без данных)

Этап 2 — Меню (2–3 дня)
  ├── Загрузка данных меню из JS-объекта (адаптация из index.html)
  ├── Рендер карточек товаров
  ├── Sticky-табы с Scrollspy
  └── Кнопка «+» / счётчик «−1+» на карточке

Этап 3 — Корзина (1–2 дня)
  ├── Bottom sheet корзины
  ├── CloudStorage для хранения состава
  ├── Прогресс-бар до бесплатной доставки
  ├── Промокод с валидацией
  └── MainButton с актуальной суммой

Этап 4 — Оформление (2–3 дня)
  ├── Экран адреса (переключатель доставка/самовывоз)
  ├── tg.requestLocation() + DaData
  ├── tg.requestContact() на экране деталей
  ├── Форма деталей (имя, оплата, комментарий)
  └── Отправка в Frontpad API

Этап 5 — Полировка (1–2 дня)
  ├── Экран «Заказ принят» с анимацией
  ├── Haptic feedback
  ├── Skeleton-загрузка
  ├── Тестирование на iOS + Android
  └── Оптимизация бандла
```
