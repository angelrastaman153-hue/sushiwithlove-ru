## Сессия 1

Тема: 3 проблемы в админке заказов — «Нет в FP», не видно позиций, колонки не как в FP

### Сделано

1. **Баг с «⚠ Нет в FP»** — нашли причину: `order.php:155` читал `$fp['id']`, а FrontPad API возвращает id в поле `order_id`. Теперь код пробует все варианты: `order_id` → `order_number` → `id` → `number`. После фикса `link_fp.php` будет вызываться корректно → `fp_order_id` будет записываться в БД.

2. **Позиции заказа не было видно** — архитектурный пробел: orders хранил только суммы. Добавлены новые поля в БД через `api/migrate_items_json.php`:
   - `items_json TEXT` — массив позиций
   - `delivery_type VARCHAR(20)` — delivery / self
   - `address VARCHAR(255)` — адрес или «Самовывоз»
   - `pay_type VARCHAR(20)` — cash / qr
   - `comment TEXT` — комментарий клиента
   - `save.php` теперь сохраняет эти поля + список позиций

3. **Колонки админки под FrontPad** — заголовки: № / FP · Время · Клиент · Телефон · Адрес · Позиции · Сумма · Оплата · Статус · Действия
   - Добавлена кнопка «📋 N поз.» — открывает модалку с полным составом заказа (ST/QR/цена, скидка, баллы, комментарий)
   - Бейдж «ТЕСТ» для is_test=1
   - Сумма расписана (доставка, промо, баллы)
