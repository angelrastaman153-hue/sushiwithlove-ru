## 1. Конкуренты и примеры

### 1.1 Яндекс.Еда / Delivery Club — эталонный стандарт

**Что цепляет:**
- Полноэкранный баннер с аппетитными фотографиями, строка поиска адреса по центру
- Яндекс.Еда: жёлто-чёрная гамма `#FFDD00` + `#1A1A1A`
- Delivery Club: малиново-белая `#FF2D55` + `#FFFFFF`

**Секции:** поиск адреса → категории → акции → приложение → отзывы

**Что взять:**
- Строку поиска с геолокацией как главный CTA
- Карточки с рейтингом, временем доставки и минимальной суммой заказа
- Липкую шапку с корзиной

---

### 1.2 Две Палочки

**Что цепляет:** тёмный фон, крупная фотография суши, премиальный японский ресторан

**Цвет:** `#1C1C1C` + `#C9A96E` (золото) + `#FFFFFF`

**Секции:** меню с фильтрами → акции дня → зоны доставки → лояльность → отзывы → блог

**Что взять:**
- Золотые акценты на тёмном фоне = воспринимаемое «качество»
- Карточки блюд с КБЖУ — доверие у ЗОЖ-аудитории
- Зонирование доставки прямо на странице

---

### 1.3 Тануки

**Что цепляет:** видео-фон / слайдер с профессиональным фото, двойной CTA «Доставка / Рестораны»

**Цвет:** `#8B0000` (тёмно-красный) + `#1A1A1A` + `#F5F5F5`

**Что взять:**
- Переключатель режимов «доставка / зал» на главном экране
- Программа лояльности с видимым счётчиком баллов
- Видео-герой с профессиональной едой

---

### 1.4 Суши Wok — агрессивная конверсия

**Что цепляет:** яркий светлый дизайн, крупные плашки акций, таймер обратного отсчёта, FOMO

**Цвет:** `#E8000D` (красный) + `#FFD700` + `#FFFFFF`

**Что взять:**
- Таймер обратного отсчёта на акции — мощный FOMO
- «Быстрое добавление» в корзину одной кнопкой без открытия карточки
- Блок «Хиты продаж» для увеличения среднего чека

---

### 1.5 Евразия

**Цвет:** `#2B2B2B` + `#D4AF37` (золото) + `#F0F0F0`

**Что взять:**
- Блок «Забронировать стол» — расширяет аудиторию за пределы доставки
- Раздел «Банкеты» как дополнительный канал дохода

---

### 1.6 Якитория — японский минимализм

**Что цепляет:** много белого пространства, аутентично японский, сезонные предложения

**Цвет:** `#FFFFFF` + `#333333` + `#B22222` (японский красный) + бежевые акценты

**Что взять:**
- Сезонные предложения для удержания аудитории
- Белое пространство для премиального ощущения
- Японская типографика как элемент аутентичности

---

### 1.7 Региональный сегмент (Tilda-лендинги)

**Типичная структура:**

| Блок | Содержание |
|------|-----------|
| Hero | Фото роллов + «Доставка за 60 минут» + кнопка |
| Акции | 2–3 баннера (1+1, скидка 20%, сет дня) |
| Меню | Табы по категориям + карточки с фото и ценой |
| Доставка | Условия + карта зон + минимальная сумма |
| О нас | 3–4 иконки с преимуществами |
| Отзывы | Слайдер или сетка |
| CTA | Форма заказа или кнопка WhatsApp/Telegram |
| Футер | Телефон, соцсети, адрес |

**Что взять:** CTA в Telegram/WhatsApp работает лучше форм в регионах

---

### 1.8 Концепты Behance — российские дизайнеры

**Dark Sushi Landing:**
- Фон `#0D0D0D`, акцент `#FF6B35` (оранжевый) или `#00FFB2` (неоновый мятный)
- Еда в технике «floating food» (без подложки на тёмном фоне)
- Параллакс-скролл, micro-анимации при наведении на карточки

**Minimalist Sushi UI:**
- Много белого, роллы как главный визуал
- Сетка 3×N, hover с увеличением и кнопкой «В корзину»
- Палитра: `#FAF9F6` (молочный) + `#2C2C2C` + `#E63946` (красный)

---

### Сводная таблица по сегментам

| Параметр | Массовый | Средний | Премиум |
|----------|---------|---------|---------|
| Фон | Тёмный | Тёмный/смешанный | Светлый тёплый |
| Акцент | Красный/жёлтый | Золото/оранжевый | Нейтральный бежевый |
| Главный CTA | Телефон/WhatsApp | Онлайн-корзина | Бронирование |
| Анимации | Нет | Базовые CSS | Параллакс, видео |
| Фото еды | Стоковые | Собственные | Студийные |

---

### Топ ошибок конкурентов

1. Стоковые фото еды — аудитория распознаёт, теряет доверие
2. Перегруженный первый экран (3+ CTA одновременно)
3. Отсутствие мобильной оптимизации
4. Японские иероглифы как «декор» без смысла — выглядит дёшево
5. Нет КБЖУ/состава — критично для ЗОЖ-аудитории

## 2. Структура лендинга — 9 блоков

> Главная задача: довести посетителя от первого взгляда до «Оформить заказ» с минимальным трением.  
> Конверсия food delivery лендингов: 3–8% холодный трафик, 10–20% тёплый.

---

### Блок 1 — Hero (Первый экран)

**Цель:** ответить за 3–5 секунд: кто вы, что предлагаете, почему нужно заказать сейчас.

| Элемент | Описание |
|---------|---------|
| Логотип | Левый верхний угол, небольшой |
| Главный заголовок | «Суши и роллы с доставкой за 45 минут по Кургану» |
| Подзаголовок | «Свежая рыба каждый день. Минимальный заказ 500 ₽» |
| CTA-кнопка | «Перейти к меню» — крупная, контрастная |
| Фон | Аппетитное фото/видео роллов — живое, не стоковое |
| Акционный бейдж | «Бесплатная доставка от 800 ₽» или «−20% на первый заказ» |
| Телефон | В шапке, кликабельный (tel:) |
| Время работы | «Доставка с 10:00 до 22:00» |

**Работает лучше:** фото настоящей еды (CTR +20–35%), кнопка заказа без скролла, таймер обратного отсчёта (+8–15% конверсии).

---

### Блок 2 — Полоса доверия

**Быстрые социальные доказательства сразу под hero:**

```
[★ 4.8 на Яндекс.Картах]  [2 300+ заказов/мес]  [Доставка 45 мин]  [С 2019 года]
```

4 иконки с цифрами. Реальные данные из Яндекс.Карт / 2ГИС — доверие выше.

---

### Блок 3 — Акции и спецпредложения

**Акции — главный триггер принятия решения в доставке еды в России.**

| Элемент | Описание |
|---------|---------|
| Карточки акций | 2–4 шт. с визуалом, условием и CTA |
| Постоянные акции | «Бесплатная доставка от 900 ₽», «Комбо на двоих −15%» |
| Временные акции | «Только до пятницы: мисо-суп в подарок» |
| Таймер | Для ограниченных акций — таймер обратного отсчёта |
| Промокод | Поле ввода, связанное с корзиной |

---

### Блок 4 — Меню с корзиной ← главный блок

**Sticky навигация по категориям:**
```
[Популярное] [Роллы] [Сеты] [Суши] [WOK] [Напитки] [Соусы]
```

**Карточка блюда:**

| Элемент | Описание |
|---------|---------|
| Фото | Качественное, 4:3, нейтральный фон |
| Название | Читаемое, без лишних иероглифов |
| Состав | «Лосось, авокадо, огурец, сливочный сыр» |
| Вес | «260 г / 8 шт» |
| Цена | Крупно, рядом с кнопкой |
| Кнопка | «В корзину» → счётчик «+/−» |
| Бейдж | «Хит», «Новинка», «Острое» |

**Корзина:**
- Плавающая кнопка (sticky) в правом нижнем углу на мобиле
- Slide-out drawer на десктопе
- Bottom sheet на мобиле

---

### Блок 5 — Оформление заказа

**Правило:** не более 5–6 полей, каждое лишнее поле −3–5% конверсии.

**Поля:**

| Поле | Обязательность |
|------|---------------|
| Телефон | Обязательно (главный ID в Frontpad) |
| Имя | Желательно |
| Улица + дом | Обязательно |
| Квартира/офис | Опционально |
| Способ оплаты | Наличные / Карта курьеру / Онлайн |
| Комментарий | Опционально |

**UX:** автодополнение улиц (DaData), маска на телефон, прогресс-бар шагов.

---

### Блок 6 — Доставка и зоны

**Частая причина брошенной корзины — пользователь не знает, доставляют ли к нему.**

| Элемент | Описание |
|---------|---------|
| Карта | Яндекс.Карты с цветными зонами |
| Таблица зон | Центр / Заречный / Рябково → стоимость → время |
| Порог доставки | «Бесплатно от 900 ₽» — выделить ярко |
| Самовывоз | Адрес + схема проезда |

**Пример таблицы:**

| Зона | Стоимость | Время | Мин. заказ |
|------|----------|-------|-----------|
| Центр | Бесплатно | 30–40 мин | 600 ₽ |
| Заречный | 100 ₽ | 50–60 мин | 800 ₽ |
| За кольцевой | 150 ₽ | 60–75 мин | 1 000 ₽ |

---

### Блок 7 — Отзывы

**80% покупателей читают отзывы перед заказом еды (Яндекс).**

| Элемент | Описание |
|---------|---------|
| Карточки | Имя + аватар + текст + дата + звёзды |
| Источник | Иконка Яндекс / 2ГИС / VK |
| Слайдер | 5–10 отзывов горизонтально |
| Рейтинг | Крупно: ★ 4.8 · 320 отзывов |
| Фото | Фото от клиентов — если есть, обязательно |

Отзывы с фото конвертируют на 25% лучше текстовых. Отзывы не старше 6 месяцев.

---

### Блок 8 — О нас

**Краткий блок, снимающий возражение «а кто вы вообще такие?»**

| Элемент | Описание |
|---------|---------|
| История | 2–3 предложения |
| Фото кухни | Живые, не стоковые. Повара за работой |
| Преимущества | 3–5 иконок: «Свежая рыба», «Своя доставка», «Без консервантов» |
| Соцсети | ВКонтакте, Telegram |

---

### Блок 9 — Футер

| Элемент | Описание |
|---------|---------|
| Телефон | Кликабельный, крупный |
| Мессенджеры | WhatsApp / Telegram кнопки |
| Адрес | Если есть самовывоз/зал |
| Часы работы | Полное расписание |
| Политика конфиденциальности | Обязательно по 152-ФЗ |
| Юридическое лицо | ИП/ООО + ИНН — повышает доверие |
| Соцсети | ВКонтакте, Telegram |

---

### Приоритеты разработки

| Приоритет | Что делать в первую очередь |
|-----------|----------------------------|
| P0 (критично) | Меню + корзина + форма + Frontpad |
| P1 (важно) | Hero + акции + зоны доставки |
| P2 (нужно) | Отзывы + полоса доверия |
| P3 (желательно) | О нас + богатый футер |

### Что НЕ нужно на лендинге

- Слайдер в hero с 5+ слайдами (снижает конверсию)
- Сложная анимация при загрузке
- Требование регистрации перед заказом
- CAPTCHA на форме заказа
- Более 6 полей в форме
- Pop-up при входе на сайт

## 3. Тренды веб-дизайна 2025–2026

### Тренд 1 — Scroll-Triggered Animations (GSAP + ScrollTrigger)

**Описание:** анимации при скролле — стандарт premium-сайтов. Тренд сместился к narrative scroll, когда скролл разворачивает историю бренда.

**Применение к суши-лендингу:**
- Hero: тарелка с роллом «летит» на центр при скролле
- Секция «Как готовим»: 3 шага появляются поочерёдно
- Счётчики (заказы, отзывы) — animate на число при входе в viewport

**Инструменты:** GSAP + ScrollTrigger, Framer Motion, AOS  
**Сложность:** Средняя | **Приоритет:** Высокий

---

### Тренд 2 — Micro-interactions и Haptic Feedback UI

**Описание:** мельчайшие анимированные отклики на каждое действие пользователя.

**Применение:**
- Кнопка «+» в корзину: иконка суши прыгает + пульсация счётчика
- Кнопка «Заказать»: loading spinner → галочка → «Принято!»
- Промокод: success-state с micro-анимацией при применении

**Сложность:** Лёгкая/Средняя | **Приоритет:** Высокий

---

### Тренд 3 — Тёмная тема + тёплый неон

**Рекомендуемая палитра для суши-лендинга:**

| Цвет | HEX | Применение |
|------|-----|-----------|
| Основной фон | `#0F0B08` | Тёплый почти-чёрный, как соевый соус |
| Акцент 1 | `#E8A847` | CTA-кнопки, активные элементы |
| Акцент 2 | `#C84B31` | Цены, badges, промо |
| Карточки | `#1E1810` | С border `rgba(232,168,71,0.15)` |
| Glow hover | — | `box-shadow: 0 0 20px rgba(232,168,71,0.3)` |

Поддержка `prefers-color-scheme` — автоматическое переключение.  
**Сложность:** Лёгкая | **Приоритет:** Критичный

---

### Тренд 4 — Display-типографика и Kinetic Text

**Рекомендуемые шрифты (Google Fonts, бесплатно):**

| Роль | Шрифт |
|------|-------|
| Заголовок | Cormorant Garamond / Playfair Display |
| Акцент | Dancing Script / Satisfy |
| UI/текст | Inter / Manrope (кириллица) |

**Приёмы:**
- Заголовок как маска поверх фото роллов (`background-clip: text`)
- Два размера веса в одном заголовке: «СУШИ» жирное + «с душой» тонкое
- Kinetic: вес шрифта анимируется от 100 до 900 при скролле

**Сложность:** Лёгкая/Средняя | **Приоритет:** Высокий

---

### Тренд 5 — Custom Cursor + Parallax 3D

**Применение:**
- Курсор-follower: круг 12px `#E8A847` с lag-следованием
- Hover на карточку блюда: курсор → «СМОТРЕТЬ»
- Magnetic CTA: кнопка «Заказать» притягивает курсор в радиусе 60px
- Parallax hero: палочки для суши двигаются быстрее фона

⚠️ Только за `@media (hover: hover) and (pointer: fine)` — на мобиле недоступно.

**Сложность:** Средняя | **Приоритет:** Средний

### Тренд 7 — Mobile Thumb Zone + Bottom Sheet

**Bottom Sheet для корзины/карточки:**
```css
.bottom-sheet {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  border-radius: 20px 20px 0 0;
  padding-bottom: env(safe-area-inset-bottom);
  transform: translateY(100%);
  transition: transform 0.35s cubic-bezier(0.32, 0.72, 0, 1);
}
.bottom-sheet.open { transform: translateY(0); }
```

**Правила:**
- Sticky Bottom Bar на мобиле: `[🛒 3 товара  830₽  → ЗАКАЗ]`
- Touch target кнопок: минимум 48×48px
- Swipe-to-dismiss для закрытия bottom sheet

**Сложность:** Средняя | **Приоритет:** Критичный

### Тренд 8 — Dark Glassmorphism + Bento Grid

**Dark Glass карточка:**
```css
.glass-card {
  background: rgba(15, 11, 8, 0.65);
  backdrop-filter: blur(20px) saturate(1.5);
  border: 1px solid rgba(232, 168, 71, 0.15);
  border-radius: 16px;
  box-shadow: 0 8px 32px rgba(0,0,0,0.4),
              inset 0 1px 0 rgba(255,255,255,0.05);
```

## 4. UX корзины и оформления заказа

### 4.1 Frontpad API — техническая интеграция

**Базовый URL:** `https://app.frontpad.ru/api/new/`

**Аутентификация:** secret key в каждом запросе. **Бэкенд обязателен** — ключ не должен быть в браузере.

#### Создание заказа

```
POST https://app.frontpad.ru/api/new/order
```

**Обязательные поля:**

| Поле | Тип | Описание |
|------|-----|---------|
| `secret` | string | Секретный ключ заведения |
| `product[]` | array | Массив ID товаров |
| `product_kol[]` | array | Количество каждого товара |
| `name` | string | Имя клиента |
| `phone` | string | Телефон (7XXXXXXXXXX) |

**Важные опциональные поля:**

| Поле | Описание |
|------|---------|
| `street`, `home`, `apart`, `et`, `pod` | Адрес доставки |
| `pay` | 1=наличные, 2=карта курьеру, 3=онлайн |
| `datetime` | Время доставки (YYYY-MM-DD HH:MM) |
| `product_mod[]` | Модификаторы параллельным массивом |
| `score` | Бонусы для списания |
| `certificate` | Промокод |
| `self` | 0=доставка, 1=самовывоз |
| `descr` | Комментарий к заказу |
| `person` | Количество персон (палочки) |

**Ответ:**
```json
{ "order_id": "12345", "order_number": "А-101", "status": 1 }
```

#### Другие эндпоинты

```
GET /api/new/products?secret=KEY     — каталог товаров
GET /api/new/categories?secret=KEY   — категории меню
GET /api/new/order?secret=KEY&order_id=12345  — статус заказа
GET /api/new/delivery?secret=KEY     — зоны доставки (полигоны)
POST /api/new/score { phone }        — баланс бонусов клиента
```

**Особенности:**
- Нет webhook-ов — нужен polling или собственный WebSocket
- Телефон строго 11 цифр, начинается с 7
- Модификаторы — параллельный массив, пустая строка если нет
- Каталог кешировать на стороне сайта (TTL 15–60 мин)

### 4.7 Типичные ошибки

**Критические:**
- Кнопка «Оформить» не видна без скролла → Sticky CTA
- Нет валидации адреса → DaData + проверка зоны
- Нет маски телефона → неверный формат для Frontpad
- Корзина сбрасывается при перезагрузке → localStorage

**Ошибки интеграции Frontpad:**
- Не кешировать каталог → медленная загрузка
- Не обрабатывать ошибки API → «завис» заказ
- Не синхронизировать стоп-лист → товары которых нет
- Телефон в неверном формате → ошибка API

### 5.2 Почему Next.js

- Встроенные API Routes = не нужен отдельный сервер для Frontpad прокси
- SSG/ISR для SEO (Яндекс индексирует лучше SPA)
- `next/image` — автоконвертация в WebP/AVIF, lazy load, оптимизация
- Incremental Static Regeneration — меню обновляется без полного ребилда

### 5.3 Frontpad интеграция — архитектура

```
Браузер  →  Next.js API Route  →  Frontpad API
```

Secret key хранится только на сервере (env variable). Никогда не в браузере.

```typescript
// pages/api/order.ts
export default async function handler(req, res) {
  const response = await fetch('https://app.frontpad.ru/api/new/order', {
    method: 'POST',
    body: JSON.stringify({ secret: process.env.FRONTPAD_SECRET, ...req.body })
  });
  const data = await response.json();
  res.json(data);
}
```

### 5.4 Меню — хранение данных

**Вариант A (простой старт):** JSON-файл в репозитории + ручное редактирование  
**Вариант B (клиент редактирует сам):** Sanity CMS + ISR в Next.js  
**Вариант C (Frontpad как источник истины):** синхронизация через API каждые 15 мин

**Рекомендация для старта:** Вариант A → Вариант B при росте.

### 5.5 Хостинг

**Основной сервер:** Timeweb Cloud VPS (~1 000 ₽/мес) — российский IP, низкий ping  
**CDN + защита:** Cloudflare бесплатный план  
**Важно:** Russian CDN для видео и Lottie (Selectel / Yandex Cloud) — меньше latency

### 5.6 SEO для локального бизнеса

- **Яндекс.Бизнес** — критически важен для локального SEO в России
- Мета-теги: «суши доставка Курган», «роллы Курган», «заказать суши Курган»
- Core Web Vitals: LCP < 2.5 с, FID < 100 мс, CLS < 0.1
- Schema.org `LocalBusiness` + `Restaurant` разметка

### 5.7 Производительность

- `next/image` с `priority` для hero-фотографии
- Динамический импорт корзины: `dynamic(() => import('./Cart'), { ssr: false })`

## 6. Экспертная оценка проекта

> Проект: лендинг для суши-бара в Кургане с корзиной и интеграцией в Frontpad.

---

### 🎨 Эксперт 1 — UX-дизайнер (10 лет опыта)

#### Типичные ошибки новичков в структуре

**1. Герой без единственного CTA.**
Новички лепят три кнопки на первый экран: «Смотреть меню», «Позвонить», «Акции». Пользователь не знает куда кликать — не кликает никуда. Правило: **один главный призыв к действию** в hero. Для доставки еды это всегда «Перейти к меню» или «Заказать».

**2. Меню вынесено на отдельную страницу.**
Распространённейшая ошибка: лендинг красивый, а меню — ссылка на PDF или отдельный домен. Пользователь уходит и не возвращается. Меню должно быть прямо на лендинге, в том же скролле, без переходов.

**3. Корзина появляется только в конце.**
Если до кнопки «Оформить заказ» нужно доскролить до подвала — вы теряете людей. Корзина должна быть доступна в любой точке страницы: sticky-иконка в шапке, floating-кнопка на мобиле.

**4. Нет пустых состояний.**
Пустая корзина без текста, загрузка меню без индикатора, ошибка API без объяснения — пользователь думает что сайт «сломался» и уходит. Каждое состояние интерфейса должно быть продумано.

**5. Форма заказа запрашивает лишнее.**
Email (зачем?), дата рождения, отчество — всё это барьеры. Для Frontpad нужны только имя + телефон + адрес. Остальное — враги конверсии.

**6. Карточки меню без размера порции.**
«Калифорния — 350₽» — и всё. Пользователь не знает сколько штук, какой вес, хватит ли ему. Нужно: «8 шт · 240 г». Этот один факт снимает большую часть сомнений.

**7. Отсутствие feedback при добавлении в корзину.**
Нажал «+» — ничего не произошло визуально. Пользователь жмёт ещё раз, в корзине 5 одинаковых роллов. Toast-уведомление или анимация — обязательны.

---

#### Что ОБЯЗАТЕЛЬНО должно быть

| Элемент | Почему критично |
|---------|----------------|
| Sticky-шапка с корзиной и телефоном | Доступность ключевых действий в любой точке |
| Фото каждого блюда | Без фото конверсия карточки падает в 3–5 раз |
| Состав и вес в карточке | Снимает главное возражение «а что там внутри» |
| Индикатор загрузки меню | Пользователь видит что сайт работает |
| Валидация полей inline | Ошибка сразу у поля, не после нажатия «Отправить» |
| Страница «Спасибо / Заказ принят» | Без неё пользователь не знает, получили ли заказ |
| Мобильная версия с bottom bar | 60–70% заказов придёт со смартфонов |
| Карта зон доставки | Главный вопрос пользователя — «до меня доставят?» |

---

#### Что раздражает пользователей и заставляет уйти

- **Pop-up при входе** («Подпишитесь на акции!») — закрывают и уходят
- **Автовоспроизведение видео со звуком** — мгновенное закрытие вкладки
- **Бесконечный первый экран** — если нужно скролить 3 экрана до меню
- **Кнопки размером с ноготь** на мобиле — промахиваются, злятся
- **Медленная загрузка** (> 3 сек) — 40% уходит, не дождавшись
- **Неработающий промокод** без объяснения причины
- **Корзина сбросилась** после случайного закрытия вкладки
- **Нет подтверждения заказа** — «а получили ли они мой заказ?»
- **Цена изменилась** между добавлением в корзину и оформлением — шок

### 📊 Эксперт 2 — Маркетолог по конверсии

#### Оффер, который зацепит клиента

Для регионального суши-бара в Кургане работают три типа оффера:

**Оффер на скорость:**
> «Доставка роллов за 45 минут или бесплатно»

Конкретная цифра + гарантия. Моментально отстраивает от конкурентов, которые пишут «быстрая доставка».

**Оффер на первый заказ:**
> «−15% на первый заказ по промокоду КУРГАН»

Снижает барьер входа. Промокод создаёт ощущение персональной скидки, даже если он публичный.

**Оффер на качество:**
> «Охлаждённая рыба с доставкой каждый день. Без заморозки»

Бьёт в главный страх — «свежее ли?». Особенно работает на аудиторию 25–40 лет.

**Рекомендация для Кургана:** комбинировать скорость + первый заказ в hero. Качество — в блоке «О нас».

#### Триггеры доверия — полный список

**Социальное доказательство:**
- Рейтинг с Яндекс.Карт / 2ГИС — живая цифра, не нарисованные звёзды
- Счётчик заказов: «Уже 15 400 доставок по Кургану»
- Отзывы с именем, датой и источником (иконка Яндекс)
- Фото клиентов с едой (UGC) — самый сильный триггер

**Снятие рисков:**
- «Не понравится — вернём деньги» — явная гарантия качества
- Фото кухни и поваров — прозрачность производства
- «Работаем с 20XX года» — стабильность, не однодневка
- Юридическое лицо в футере — легальность бизнеса

**Срочность и дефицит:**
- Таймер обратного отсчёта на акцию («акция до 23:59»)
- «Осталось 3 порции» на популярные позиции
- «Принимаем заказы до 22:00» — видимое ограничение

**Социальность:**
- «23 заказа сделано сегодня» — живой счётчик (даже если показывается на front-end)
- Виджет или ссылка на ВКонтакте с активным сообществом

### 👤 Эксперт 3 — Владелец бизнеса (потенциальный клиент)

*«Я хочу чтобы сайт продавал, пока я готовлю роллы»*

#### Первые 5 секунд: на что смотрит владелец

Владелец суши-бара в Кургане смотрит не как UX-дизайнер и не как маркетолог. Он смотрит глазами своих клиентов и задаёт себе три вопроса:

1. **«Это похоже на правду?»** — фото настоящие или стоковые? Цены реальные? Если на сайте написано «доставка за 30 минут» — он реально может это обеспечить?

2. **«Смогу ли я сам это обновлять?»** — придёт новый ролл в меню, поменяется цена, закончится позиция. Если каждый раз нужно звонить разработчику — это проблема. Ему нужен либо простой способ редактирования, либо синхронизация с Frontpad (который он уже знает).

3. **«Не будет ли хаоса с заказами?»** — главный страх: заказы приходят на сайт, а в Frontpad их нет. Или приходят задвоенными. Интеграция должна быть надёжной — не «примерно работает», а «заказ моментально появляется в системе».

---

#### Что заставит владельца сказать «Да, хочу такой сайт»

- **Показать демо с реальным заказом**, проходящим в Frontpad — снимает главный страх
- **Мобильная версия работает идеально** — он сам закажет с телефона, проверит
- **Скорость загрузки** — он покажет сайт другу, скажет «смотри как быстро»
- **Выглядит дороже, чем стоит** — «вау, это лучше чем у Тануки в центре»
- **Просто объяснить кому угодно** — «вот меню, вот корзина, вот заказывай»

---

#### Что заставит владельца закрыть вкладку / не купить

- **«Это дорого поддерживать»** — если сайт требует разработчика для каждой мелочи
- **«Заказы задваиваются в Frontpad»** — технический баг = потеря доверия навсегда
- **«Мои клиенты не разберутся»** — если сам запутался в UX
- **«У конкурента такой же»** — типовой Tilda-шаблон без уникальности
- **«На мобиле всё мелкое»** — он сразу проверит с телефона, как большинство его клиентов
- **Медленно грузится** — «у меня клиенты на медленном интернете в спальных районах»
- **Нет цены доставки** — самый частый звонок от клиентов: «а сколько доставка?»

---

#### Скрытые ожидания владельца (о которых он не скажет вслух)

| Ожидание | Как закрыть |
|----------|------------|
| «Хочу чтобы заказов стало больше» | Показать конверсионные элементы: акции, FOMO, прогресс-бар |
| «Не хочу постоянно платить разработчику» | Синхронизация с Frontpad = меню всегда актуальное само |
| «Хочу выглядеть серьёзно» | Дизайн уровня сетевых брендов, не Tilda-шаблон |
| «Боюсь технических проблем» | Надёжная обработка ошибок API, план на случай недоступности Frontpad |
| «Хочу знать что происходит» | Яндекс.Метрика с понятными целями, не просто «трафик» |

### Общий список рекомендаций

На основе трёх экспертных точек зрения — итоговый список в порядке приоритета:

#### Критично (без этого сайт не работает)

1. **Один CTA в hero** — «Перейти к меню», крупный, контрастный
2. **Меню прямо на лендинге** — не ссылка, не PDF, а интерактивные карточки
3. **Корзина доступна везде** — sticky в шапке + floating на мобиле
4. **Надёжная интеграция с Frontpad** — через бэкенд-прокси, с обработкой ошибок
5. **Страница подтверждения заказа** — номер заказа + «ожидайте звонка»
6. **Мобильная версия с bottom bar** — большая кнопка, большой шрифт, bottom sheet
7. **Фото каждого блюда** — без фото конверсия карточки падает в разы
8. **Состав + вес в карточке** — «8 шт · 240 г · Лосось, крем-сыр, огурец»
9. **Карта зон доставки** — ответить на главный вопрос до того, как он задан
10. **Автодополнение адреса** (DaData) — без этого 15–20% бросают форму

#### Важно (влияет на конверсию)

11. **Оффер на первый экран** — скорость доставки с гарантией или скидка на первый заказ
12. **Прогресс-бар до бесплатной доставки** — увеличивает средний чек на 15–25%
13. **Рейтинг из Яндекс.Карт / 2ГИС** — живая цифра, не нарисованные звёзды
14. **Сохранение корзины** в localStorage — корзина не сбрасывается при перезагрузке
15. **Toast при добавлении** в корзину — feedback что действие сработало
16. **Таймер на акции** — FOMO работает даже в регионах
17. **Онлайн-оплата** (ЮКасса) — 55% пользователей предпочитают платить онлайн
18. **Политика конфиденциальности** — обязательна по 152-ФЗ, форма собирает персданные

#### Усиливает (делает сайт лучше конкурентов)

19. **Тёмная тема + золотой акцент** — воспринимаемое качество выше среднего рынка
20. **Собственные фото кухни и поваров** — убивает сомнение «а вдруг это перекупщик»
21. **Scroll-анимации** (GSAP) — ощущение «дорогого» сайта
22. **Upsell в корзине** — «к этому заказу часто добавляют: соус васаби, имбирь»
23. **Динамический hero** по времени суток — обеденный оффер vs вечерний
24. **Предзаполнение формы** для повторных клиентов — заказ за 2 минуты
25. **Fuzzy-поиск по меню** (Fuse.js) — «лосось» находит все блюда с лососем

---

#### Красные флаги — чего НЕ делать

| Ошибка | Последствие |
|--------|------------|
| Pop-up при входе | Мгновенный уход |
| Видео со звуком автоплей | Закрытие вкладки |
| Стоковые фото еды | Потеря доверия |
| 3+ CTA на первом экране | Паралич выбора |
| Форма с 8+ полями | Брошенный заказ |
| Меню только ссылкой / PDF | Пользователь не вернётся |
| Нет подтверждения заказа | «А получили ли они?» + звонки |
| Secret key Frontpad в браузере | Дыра в безопасности |
| Корзина сбрасывается при F5 | Раздражение, уход |
| Нет цены и условий доставки | Самый частый вопрос в чат |

## 7. Анализ конкурентов — апрель 2026 (агентное исследование)

> Исследование проведено параллельными агентами 2026-04-12.
> Источники: базы знаний об агрегаторах и сетях (до августа 2025), Baymard Institute, Nielsen Norman Group, CRO-практики food e-commerce.

---

### 7.1 Яндекс.Еда — детальный UX-разбор

#### Сильные стороны
1. **Живой трекер заказа** — карта с курьером, этапы «собирается / в пути / у двери». Снижает тревогу, пользователь не звонит в ресторан.
2. **Фото на каждую позицию** — крупные, с единым стилем. Визуальный выбор работает быстрее текстового.
3. **Быстрое повторение заказа** — «Заказать снова» в истории, всё добавляется одним нажатием.
4. **Умный поиск с опечатками** — «роллы с лосём» находит «роллы с лососем».
5. **Промокоды прямо в интерфейсе** — баннер в шапке ресторана, применяются автоматически.
6. **Корзина не очищается** — сохраняется на сервере, восстанавливается при возврате.

#### Слабые стороны
1. Комиссия агрегатора видна только при оформлении — неприятный сюрприз.
2. Меню может быть неактуальным — стоп-лист не синхронизирован.
3. Поддержка: 10–20 минут до живого оператора.
4. Минимальная сумма скрыта до момента оформления.
5. Переизбыток информации на главной — баннеры, промо, подборки.

#### Что берём
- Промокод с видимым применением и мгновенным пересчётом суммы
- Кнопка «Повторить последний заказ» при возврате (из localStorage)
- Сохранение всего состава корзины в localStorage (у нас уже сохраняется тип доставки — расширить)

---

### 7.2 Delivery Club / СберМаркет Еда

#### Сильные стороны
1. **Якорная навигация** — горизонтальный скролл категорий прикреплён к верху, активная подсвечивается.
2. **Информация до входа в меню** — время доставки, минимальная сумма, стоимость доставки.
3. **Модификаторы прямо в карточке** — без перехода на отдельный экран.
4. **Плавающая кнопка «Корзина — X₽»** — видна всегда, обновляется в реальном времени.
5. **Апселл «часто берут вместе»** — напитки и соусы при добавлении ролла.
6. **Прозрачный чекаут** — разбивка: блюда / доставка / скидка / итого. Нет сюрпризов.

#### Слабые стороны
1. Не работает в малых городах (до 200 тыс. жителей).
2. Принудительная авторизация через СберID отпугивает.
3. Поиск по меню конкретного ресторана отсутствует.

#### Что берём
- Время доставки + минимальная сумма + условия — на первом экране
- Прозрачная разбивка суммы в корзине
- Плавающая кнопка корзины с актуальной суммой

---

### 7.3 Тануки

#### Сильные стороны
1. **Sticky-навигация с Scrollspy** — активный раздел подсвечивается при прокрутке.
2. **Бейджи на фото карточек** — Хит, Новинка, Острое — без чтения описания.
3. **Модальное окно товара** — оверлей, пользователь остаётся в контексте меню.
4. **КБЖУ в аккордеоне** — для ЗОЖ-аудитории, не мешает остальным.
5. **Умный апселл** — конкретные соусы и напитки, не абстрактное «вам может понравиться».

#### Слабые стороны
1. Студийные «пластиковые» фото — живые вызывают больше доверия.
2. Форма заказа многошаговая, перегружена на мобиле.
3. Программа лояльности плохо объяснена на входе.

#### Что берём
- **Бейджи «Хит» на фото** — CSS `::before`, ~10 строк
- **Scrollspy**: подсветка активного раздела через IntersectionObserver

---

### 7.4 Суши Wok

#### Сильные стороны
1. **Фильтры-теги** — «Вегетарианское», «Острое», «До 300 ккал» — мгновенная фильтрация без перезагрузки.
2. **Счётчик +/– прямо на карточке** — не нужно идти в корзину для изменения количества.
3. **Корзина-drawer** — выезжает справа, пользователь не теряет контекст меню.
4. **«Часто берут вместе» в корзине** — 2–3 мини-карточки с кнопкой «+ добавить».

#### Слабые стороны
1. Мобильные карточки мелкие на экранах до 5 дюймов.
2. Поиск по меню скрыт — новые пользователи не находят.
3. Баннеры занимают 40% главной — до меню надо скроллить.

#### Что берём
- **Счётчик +/– прямо на карточке** (кнопка «В корзину» превращается в `[−] 1 [+]`)
- **Апселл внутри корзины** (у нас блок 7 «часто берут вместе» — продублировать в корзину)

---

### 7.5 2 Берега

#### Сильные стороны
1. **Прогресс-бар до бесплатной доставки** — «Добавьте ещё 200₽» — один из сильнейших стимулов увеличить чек.
2. **Промокод для новых клиентов в шапке** — снижает барьер первого заказа.
3. **Акции с таймером обратного отсчёта** — urgency без агрессии.
4. **Сеты по сценарию** — «обед на двоих», «корпоративный», «детский».

#### Слабые стороны
1. Много текста на странице акции — мобильные читают только 2 строки.
2. Регистрация обязательна для заказа — главный убийца конверсии.
3. Карточка не показывает похожие позиции.

#### Что берём
- **Прогресс-бар до бесплатной доставки** — ~20–30 строк JS в корзине
- **Таймер на акционный блок** — реальный, не бесконечно сбрасывающийся

---

### 7.6 Суши-Мастер

#### Сильные стороны
1. **Фото «в разрезе»** — видна начинка, снижается страх заказать незнакомое блюдо.
2. **Кастомизация прямо в карточке** — острота, тип риса.
3. **Чёткое зонирование** — хиты / новинки / акции / меню визуально разделены.

#### Слабые стороны
1. Устаревший дизайн карточек (стиль 2018 года).
2. Баннеры не адаптированы под iOS Safari — обрезаются.
3. Нет quick view — только полное открытие страницы товара.

#### Что берём
- Фото ролла «в разрезе» как второе фото (hover / свайп)
- Quick View Modal (`<dialog>`) при клике на карточку

#### Обязательные UI-элементы

| Элемент | Почему |
|---------|--------|
| Плавающая кнопка корзины с суммой | Пользователь всегда видит, сколько набрал |
| Счётчик +/– прямо на карточке | Убирает трение при добавлении нескольких штук |
| Бейджи «Хит» / «Новинка» на фото | Решение без чтения описания |
| Прогресс-бар до бесплатной доставки | Психологический стимул добавить ещё |
| Сохранение корзины в localStorage | Возврат с уже готовой корзиной |
| `tel:` ссылка на номер телефона | Аудитория 35+ звонит, не заказывает онлайн |
| `type="tel"` на поле телефона | Правильная цифровая клавиатура на мобиле |
| Кнопки минимум 44×44px | Стандарт iOS/Android, без промахов пальцем |

### 7.8 Топ-10 фич для внедрения (по приоритету)

#### Сейчас — минимальный код, максимальный эффект
1. **Бейджи «Хит» на карточках** — CSS `::before`, ~10 строк
2. **Таймер обратного отсчёта** на акционный блок — ~20 строк JS
3. **Промокод с мгновенной валидацией** — ~30–40 строк JS (частично уже есть)

#### Ближайшая итерация
4. **Прогресс-бар до бесплатной доставки** — ~20–30 строк JS в корзине
5. **Счётчик +/– прямо на карточке** — рефакторинг кнопки добавления
6. **Фильтры-теги** — `data-tags` + `classList.toggle('hidden')`, без библиотек

#### Следующий спринт
7. **Апселл внутри корзины** — 2–3 мини-карточки «часто берут вместе»
8. **Quick View Modal** — `<dialog>` с данными из JS-объекта товара
9. **Кнопка «Повторить заказ»** — из localStorage, предзаполнить корзину
10. **Scrollspy** — IntersectionObserver, подсветка активного раздела при скролле

### 7.9 Антипаттерны — что убивает конверсию

| Ошибка | Последствие |
|--------|-------------|
| Обязательная регистрация до заказа | 24% бросают корзину (Baymard Institute) |
| Скрытая стоимость доставки | Злость и уход на этапе оформления |
| Плохие или стоковые фото | Блюдо без фото берут в 3–5 раз реже |
| Попапы при первом входе | Уход за первые 5 секунд |
| PDF-меню или меню внешней ссылкой | Нельзя добавить в корзину, не индексируется |
| 3 колонки карточек на мобиле | Фото мелкие, текст нечитаем |
| Нет обратной связи при добавлении | Пользователь не понимает, добавилось ли |
| Длинная форма чекаута (8+ полей) | Брошенный заказ |
| «Доставим быстро» без цифр | Неопределённость = звонки вместо заказа |
| Акции с вечным таймером «00:00» | Потеря доверия ко всему сайту |
